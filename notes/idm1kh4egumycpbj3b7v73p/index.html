Type De Mémoire en Java

heap memory

Mémoire principale, contenant les objets créé par l'application.

Configurable par le -Xms et -Xmx exemple java -Xms4096M -Xmx6144M ClassName

Java utilise un Garbage collector pour le nettoyage de la heap : Garbage Collector Java

Stack memory

Stock les variables locale et les informations de méthode mais aussi les thead exécution

Il n'est pas géré par les Garbage collector mais par la jvm directement.

Native Memory

L'allocation de mémoire a l'extérieur de la heap java et utilisé par la jvm. Aussi appelé off-heap memory

Utilisé pour réaliser le serialisation en lecture et écriture. Les performances dépend du buffer, du process de serialisation et de l'espace disque

La JVM stock les stack de thread, les données Internet and memory mapped files.

Direct Memory

Allocation en dehors de la java heap. Ça représente la mémoire utilisé sur l'OS par le process de la JVM.

Java NIO utilisé cette mémoire pour écrire les données sur le réseau ou le disques

On peut limiter le direct buffer memory size en utilisant ce paramètre : -XX:MaxDirectMemorySize=1024M
